# Backlog inicial priorizado de Prodexa

## Epicas por valor de negocio
1. Fundacion tecnica y arquitectura base.
2. Dominio de recetas e ingredientes.
3. Motor de costeo y simulacion de precios.
4. Seguridad y control de acceso.
5. Dashboard y experiencia de usuario.
6. Observabilidad y operacion.

## Backlog priorizado (inicio tecnico)

### Prioridad P1
- Crear estructura base fullstack (backend + frontend + db).
- Definir modelo de datos inicial de recetas e ingredientes.
- Implementar endpoints de recetas (alta, consulta, actualizacion).
- Implementar caso de uso de simulacion de costos.
- Configurar validaciones de entrada y manejo centralizado de errores.

### Prioridad P2
- Implementar autenticacion con JWT y refresh token.
- Implementar roles y permisos basicos.
- Crear vistas iniciales del dashboard con KPIs clave.
- Configurar logs estructurados y endpoint de salud.

### Prioridad P3
- Integrar reglas de promociones avanzadas.
- Agregar exportacion de reportes.
- Integrar asistencia IA local con Ollama.
- Configurar metricas y tablero operativo tecnico.

## Criterio de priorizacion
- Impacto en negocio.
- Riesgo tecnico.
- Dependencias de arquitectura.
- Esfuerzo estimado.

## Definicion de orden para ejecucion
Primero se ejecuta toda la prioridad P1, luego P2 y finalmente P3.
